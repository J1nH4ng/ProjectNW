Ansible 是一款基于 Python 开发的自动化工具，主要用于配置管理、应用部署、任务自动化和持续交付。由 Red Hat 公司维护，采用<b>无代理架构（即无需在目标主机上安装客户端）</b>，通过 SSH 协议进行通信，简化了大规模系统的管理。

## 主要特点

- 无代理架构（部署简单）：直接通过 SSH（Linux）或 WinRM（Windows）协议管理目标主机，无需安装额外的客户端。
- 基于模块化设计：Ansible 本身仅提供框架，真正的批量操作由模块实现。
- 声明式 `YAML` 语法：使用 Playbook 定义自动化任务，直观易读，方便完成复杂任务。
- 跨平台支持：各平台设备都可以被 Ansible 管理。
- 可拓展：支持 API 及自定义模块，可以通过编写 Python 文件进行拓展。
- 幂等性：重复执行任务不会导致系统状态异常，确保操作的一致性。

## 工作原理

Ansible 本身类似于一个工具箱，其中存放了许许多多的工具，及 Ansible 的模块，根据不同的场景需求，我们需要选择最合适的工具（模块）。而 Ansible 的功能都由这些模块来实现，其执行过程如下所示：

<Steps>
<Step>
用户发出 Ansible 命令（Ad-Hoc 或 Playbook）
</Step>

<Step>
Ansible 主程序加载自己的配置文件 `/etc/ansible/ansible.cfg`
</Step>

<Step>
读取主机清单中的设备 IP 或域名以及变量
</Step>

<Step>
调用 Ansible 命令中指定的模块，通过 Ansible 将模块参数生成对应的临时 python 脚本，并传输至目标服务器的临时目录
</Step>

<Step>
对应目标主机的执行用户的临时目录中出现该 python 脚本文件，给予文件可执行权限，执行该脚本并返回结果。
</Step>

<Step>
删除临时 python 脚本文件并退出。
</Step>
</Steps>
